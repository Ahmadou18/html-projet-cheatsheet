Structure de base d'un document HTML

<!doctype html>
<html>
    <head></head>
    <body></body>
</html>

<select></select>
   <La balise select permet de créer une liste déroulante ou à choix multiples>
    

<optgroup></optgroup>
    <La balise optgroup est utilisée pour grouper au sein de la balise "select" un
    ensemble d'éléments options de même type >
   

<options></options>
    <La balise "option" s'utilise spécifiquement à l'intérieur d'une liste :
    "select". Elle définit un élément de cette liste >

exemple :


    <select class="" name="select">
        <optgroup label="optgroup">
            <option>option1</option>
            <option>option2</option>
        </optgroup>
        <optgroup label="optgroup">
            <option>option3</option>
        </optgroup>
    </select>




    




<fieldset></fieldset>
  <la balise "fieldset" est employée pour structurer plus facilement les formulaires.
  Elle permet en effet de regrouper et d'encadrer de façon visuelle un ensemble de champs.
  Il est de plus possible d'attribuer un titre à ce groupement en employant la balise
  "legend">

<labels></label>
  <La balise "label" permet de rattacher une information textuelle à un contrôle
  par le biais d'une étiquette. Ce tag est généralement utilisé dans les formulaires
  pour mettre en relation une zone de saisie et son descriptif">

<legend></legend>
  <La balise "legend" est utilisée au sein du tag "fieldset".
  Celui-ci permet de créer un cadre autour des éléments qu'il contient,
  et la balise "legend" permet de donner un titre à ce cadre>

<form></form>
  Les balises FORM et /FORM entourent un formulaire. La balise FORM peut contenir des attributs :NAME=nom qui permet de différencier plusieurs formulaires sur une même page.
  Entre les balises FORM et /FORM on met du texte et des objets destinés à recueillir les actions des utilisateurs : ligne de saisie, boutons, listes de sélection, ...

        Exemples :
        <form>
            form elements
        <form>

<input></input>
   Les balises INPUT et /INPUT permettent d'inclure dans un formulaire des objets destinés à réagir aux actions de l'utilisateur. Les attributs de la balise INPUT permettent de préciser le nom de l'objet et le type d'objet désiré.

                Exemples :

                <form>

                First name:


                <input type="text" name="firstname">


                Last name:


                <input type="text" name="lastname">

                </form>

                


                    First name:


                    


                    Last name:


                    

                


<textarea></textarea>
Les balises TEXTAREA et /TEXTAREA permettent de définir une zone de saisie à plusieurs lignes.

                Exemples :

                <textarea name="comment" form="usrform">Enter text here...</textarea>

                

<button></button>

La balise permet de créer un contrôle de type bouton. C'est l'un des différents éléments utilisables au sein d'un formulaire <form> (au même titre que select, textarea et input).
Il est possible de placer du code html à l'intérieur de l'élément button.

                Exemples :

                <button type="button">Click Me!</button>

                





<datalist>


   
La balise <datalist> permet de définir une liste de valeurs qui sont associées à un élément de type <input> pour proposer un mécanisme d'auto complétion lors de la saisie de texte par l'utilisateur. Les valeurs ajoutées dans la balise <datalist> sont encadrées par des éléments <option>



 
<keygen></keygen>


   
La balise <keygen> permet de générer au sein d'un formulaire une paire de clés de sécurité pour permettre le cryptage et le décryptage de données échangées. La clé privée est conservée en local tandis que la clé publique est retournée au serveur. 




<output></output>


  
La balise <output> représente le résultat d'un calcul.
