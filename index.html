Basic HTML

p

La balise <p> permet de créer un paragraphe.

Exemple :
<p>Ceci est un paragraphe.<p>

Paramètres d'affichage CSS par défaut :
La plupart des navigateurs vont afficher l'élément <p> avec les valeurs suivantes : 

 
                     p {
                        display: block;
                        margin-top: 1em;
                        margin-bottom: 1em;
                        margin-left: 0;
                        margin-right: 0;
                       }

Images

<img>
L'élément HTML <img>permet de représenter une image dans un document. L'attribut src est ignoré lorsque l'agent utilisateur prend en charge srcset et que ce dernier contient des descripteurs avec 'w'. Dans cet exemple, lorsque la condition (min-width: 600px) est vérifiée pour le média utilisé, la largeur de l'image sera 200px, sinon, elle occupera 50vw (ce qui correspond à 50% de la largeur de la zone d'affichage (viewport)).

Exemple

                    <img src="image.png">
<map>
L'élément HTML <map> est utilisé avec des éléments <area> afin de définir une image cliquable divisée en régions.

Exemple

    <map name="exemple-map-1">
    <area shape="rect" coords="0,0 200,200" href="https://developer.mozilla.org"
    target="_blank" />
    <area shape="default" />
    </map>
    <img usemap="#exemple-map-1" src="https://mdn.mozillademos.org/files/14546/map.png">
<area>
L'élément HTML <area> définit une zone particulière d'une image et peut lui associer un lien hypertexte. Cet élément n'est utilisé qu'au sein d'un élément <map>

Exemple

  <map name="exemple-map-1">
<canvas>
L'élément HTML <canvas> fournit une zone graphique vide sur laquel des API JavaScript spécifiques peuvent dessiner (tel que des Canvas 2D ou du WebGL).

Exemple

   <canvas id="myCanvas"></canvas>

 <script type="text/javascript">
  	var canvas=document.getElementById('myCanvas');
  	var ctx=canvas.getContext('2d');
  	ctx.fillStyle='#FF0000';
  	ctx.fillRect(0,0,80,100);
 </script>
<figcaption>
L'élément HTML <figcaption> permet d'indiquer un sous-titre, une légende, associé à une figure ou à une illustration (cette dernière étant représentée par l'élément <figure> qui est le parent direct de la légende). L'élément <figcaption> est optionnel ; s'il n'est pas présent, la figure n'aura pas de légende.

Exemple

                    <figure>
                   <figcaption>Jijy, le chat de Stéphanie<figcaption>
                   <img src="jiji.jpg" width="304" height="228" />
                </figure>
<figure>
Utilisée pour regrouper un ou plusieurs médias ( illustrations, diagrammes, exemples de code, photos...) attachés au document sans pour autant en suivre le flux. Les images s'y trouvant n'ont pas obligation d'être inséré dans une balise <p> et dans le cas d'une liste d'images, on peut omettre les listes à puce. On utilisera la balise figcaption pour décrire les médias utilisés. Confusion : cette balise ne doit être utilisée dès qu'il y a une image dans un article, on peut aussi la confondre avec la balise <aside>.

Exemple

                  <figure>
 <p>Stéphanie a un chat qui s'appelle Jiji</p>
 <img src="jiji.jpg" width="304" height="228" />
</figure>
<picture>
L'élément HTML <picture> est un conteneur utilisé afin de définir plusieurs éléments <source> destinés à un élément <img>. Le navigateur choisira la source la plus pertinente selon la disposition de la page (les contraintes qui s'appliquent à la boîte dans laquelle l'image devra être affichée) et selon l'appareil utilisé (la densité de pixels de l'affichage par exemple).

Exemple

                  <picture>
 <source srcset="mdn-logo-wide.png" media="(min-width: 600px)">
 <img src="mdn-logo-narrow.png" alt="MDN">
</picture>
<svg>
Le W3C a commencé à travailler sur SVG à la fin des années 90, mais SVG n'est devenu populaire qu'avec la sortie d'Internet Explorer 9, supportant le SVG. Tous les principaux navigateurs le prennent maintenant en charge. Basé sur une syntaxe XML, SVG peut être stylé avec CSS et être rendu interactif grâce à JavaScript. HTML5 permet à présent d'intégrer directement des balises SVG au sein d'un document HTML. SVG étant un format d'image vectorielle, les images SVG peuvent être redimensionnées à l'infini, ce qui les rend inestimables pour le responsive design, car elles permettent de créer des éléments d'interface qui s'adaptent à toutes les tailles d'écran. SVG fournit également un ensemble d'outils, comme du clipping, des masques, des filtres et des animations.

Exemple

<svg width="100" height="100">
 <circle cx="50" cy="50" r="40" stroke="green" stroke-width="4" fill="yellow" />
</svg>

Les balises listes

< ul >
La balise < ul > est utilisée pour définir une liste non ordonnée, c'est à dire une liste d'éléments non numérotés précédés de façon classique par une puce. Le rendu des listes varie suivant les navigateurs : il est utile d'appliquer des styles css pour contrôler au mieux l'affichage des listes. 
                            ul {
                                display: block;
                                list-style-type: disc;
                                margin-top: 1em;
                                margin-bottom: 1 em;
                                margin-left: 0;
                                margin-right: 0;
                                padding-left: 40px;
                                }
< ol >
La balise < ol > définit une liste ordonnée, c'est à dire une liste tenant compte de l'ordre des éléments qu'elle contient. Par défaut, chacun de ses éléments est précédé de son numéro d'apparition (elle applique implicitement l'attribut "type=1"). 
                                ol {
                                display: block;
                                list-style-type: decimal;
                                margin-top: 1em;
                                margin-bottom: 1em;
                                margin-left: 0;
                                margin-right: 0;
                                padding-left: 40px;
                                    }
< li >
La balise < li > encapsule des éléments présents dans des listes. Elle est toujours employée lors de l'utilisation de listes ordonnées, déclarées avec la balise < ol >, ou non ordonnées : balise < ul >. Les éléments < dir > et < menu > emploient également la balise < li > pour encapsuler les éléments qu'ils contiennent. 
                                li {
                                display: list-item;
                                    }
< dl >
La balise < dl > fait partie des tags permettant d'établir des listes de définitions.(dl (Definition list) : élément structurel annonçant et encadrant une liste de définitions.) 
                                dl {
                                display: block;
                                margin-top: 1em;
                                margin-bottom: 1em;
                                margin-left: 0;
                                margin-right: 0;
                                        }
< dt >
La balise < dt > fait partie des tags permettant d'établir des listes de définitions.(dt (Definition term) : élément contenant le terme à définir.) 
                                    dt {
                                    display: block;
                                        }
< dd >
La balise < dd > fait partie des tags permettant d'établir des listes de définitions.(dd (Definition description) : élément contenant la description du terme précédemment écrit.) 
                                dd {
                                    display: block;
                                    margin-left: 40px;
                                    }

Le formulaire

<select>
La balise select permet de créer une liste déroulante ou à choix multiples

<optgroup>
La balise optgroup est utilisée pour grouper au sein de la balise "select" un ensemble d'éléments options de même type

<options>
La balise "option" s'utilise spécifiquement à l'intérieur d'une liste : "select". Elle définit un élément de cette liste

exemple :



    <select class="" name="select">

        <optgroup label="optgroup">

            <option>option1</option>

            <option>option2</option>

        </optgroup>

        <optgroup label="optgroup">

            <option>option3</option>

        </optgroup>

    </select>


<fieldset>
la balise "fieldset" est employée pour structurer plus facilement les formulaires. Elle permet en effet de regrouper et d'encadrer de façon visuelle un ensemble de champs. Il est de plus possible d'attribuer un titre à ce groupement en employant la balise "legend"

<legend>
La balise "legend" est utilisée au sein du tag "fieldset". Celui-ci permet de créer un cadre autour des éléments qu'il contient, et la balise "legend" permet de donner un titre à ce cadre

Exemple :


<fieldset>
    <legend>Personalia:<legend>
       Name: <input type="text">

       Email: <input type="text">

       Date of birth: <input type="text">
<fieldset>
Personalia: Name:
Email:
Date of birth:


<labels>
La balise "label" permet de rattacher une information textuelle à un contrôle par le biais d'une étiquette. Ce tag est généralement utilisé dans les formulaires pour mettre en relation une zone de saisie et son descriptif"

Exemple :

<label for="male">Male<label>
     <input type="radio" name="gender" id="male" value="male">

<label for="female">Female<label>
     <input type="radio" name="gender" id="female" value="female">

<label for="other">Other<label>
     <input type="radio" name="gender" id="other" value="other">

     <input type="submit" value="Submit">



<form>
Les balises FORM et /FORM entourent un formulaire. La balise FORM peut contenir des attributs :NAME=nom qui permet de différencier plusieurs formulaires sur une même page. Entre les balises FORM et /FORM on met du texte et des objets destinés à recueillir les actions des utilisateurs : ligne de saisie, boutons, listes de sélection, ...



<form> form elements <form> 
    form {
    display: block;
    margin-top: 0em;
}

<input>
Les balises INPUT et /INPUT permettent d'inclure dans un formulaire des objets destinés à réagir aux actions de l'utilisateur. Les attributs de la balise INPUT permettent de préciser le nom de l'objet et le type d'objet désiré.


Exemples :



    <form>
        First name:

            <input type="text" name="firstname">

        Last name:

            <input type="text" name="lastname">
    </form>

Last name:

<textarea>
Les balises TEXTAREA et /TEXTAREA permettent de définir une zone de saisie à plusieurs lignes.

Exemples :



    <textarea name="comment" form="usrform">Enter text here...</textarea>




<button>
La balise permet de créer un contrôle de type bouton. C'est l'un des différents éléments utilisables au sein d'un formulaire <form> (au même titre que select, textarea et input). Il est possible de placer du code html à l'intérieur de l'élément button.

Exemples :



    <button type="button">Click Me!</button>


<datalist>
La balise <datalist> permet de définir une liste de valeurs qui sont associées à un élément de type <input> pour proposer un mécanisme d'auto complétion lors de la saisie de texte par l'utilisateur. Les valeurs ajoutées dans la balise <datalist> sont encadrées par des éléments <option>



<keygen>
La balise <keygen> permet de générer au sein d'un formulaire une paire de clés de sécurité pour permettre le cryptage et le décryptage de données échangées. La clé privée est conservée en local tandis que la clé publique est retournée au serveur.
